# Git Workflow — ToDoBetter

Personal reference for a clean, professional solo development flow.

## Branch Naming

Use prefixes to categorize work:

```
feature/kanban-reorder     # New feature
fix/cursor-issue           # Bug fix
refactor/api-cleanup       # Code cleanup
docs/readme-update         # Documentation
chore/deps-update          # Dependencies, config
```

## When to Branch

- **Branch if**: Working on a feature/fix that takes >30 minutes
- **Skip branch if**: Quick fix (<5 min) — commit straight to main
- **Always branch if**: You're unsure if the change will work

## Commit Message Format

```
Verb: what you did + why it matters

Example:
  Add drag-and-drop reordering for tasks
  Fix cursor:pointer not applying on buttons
  Refactor API routes into separate files
  Update README with mobile layout notes
```

**Good verbs**: Add, Fix, Refactor, Update, Remove, Implement, Improve, Optimize

**Bad commits**: "misc", "fixes", "wip", "asdf", "lol"

## End-of-Session Workflow

```bash
# 1. See what changed
git status
git diff --stat

# 2. Stage and commit
git add .
git commit -m "Verb: description"

# 3. Push to remote
git push origin branch-name

# 4. If on feature branch, create PR
gh pr create --title "Short description" --body "Details"

# 5. Test in preview, then merge
gh pr merge PRNUM --squash --delete-branch

# 6. Back to main
git checkout main
git pull origin main
```

## Avoid Branch Chaos

- Always push before closing the terminal (backup to GitHub)
- Merge feature branches immediately after testing (don't let them rot)
- Delete merged branches with `--delete-branch` flag or manually with `git branch -D name`
- Main branch is always deployable — if it's not ready, don't merge

## Useful Commands

```bash
# See all local branches
git branch

# See all remote branches
git branch -a

# Delete a local branch
git branch -D feature/old-thing

# See what's on origin vs local
git fetch origin
git log --oneline main..origin/main  # commits on remote not local
git log --oneline origin/main..main  # commits on local not remote

# Undo last commit (keep changes)
git reset --soft HEAD~1

# Undo last commit (discard changes)
git reset --hard HEAD~1
```

## The 90-Second Rule

If you can't explain the commit in one sentence, it's too big. Break it into smaller commits.

**Session Ritual**: Branch → Code → Commit → Push → PR → Merge → Done
